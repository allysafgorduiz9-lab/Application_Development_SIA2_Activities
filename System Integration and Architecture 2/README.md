Description of the Repository: This repository contains all activities and projects completed for SIA2


List and explanation of the activities:

Activity 1(UML-based title proposal): The Simeon Brewers Coffee: Web-Based Ordering and Administration System is an individual activity where I designed and proposed a system based on UML concepts. The project focuses on developing a coffee ordering and administration system that allows customers to place orders online while enabling administrators to manage products and transactions. In this activity, I created a professional project title, defined the system’s purpose and scope, identified the key users and components, and developed a preliminary UML diagram to represent the system. The output demonstrates planning, system analysis, and proper application of UML design principles for a real-world web-based system.

Activity 2(sk-event-list): “SK Event List” using Laravel is an individual mini-system that displays and manages Sangguniang Kabataan (SK) events. It is a web application that allows users to view a list of SK events and check detailed information for each selected event. It provides a simple and organized navigation where users can browse events from the list page, open a specific event for more details, and return back to the main list page.

Activity 3(youth-suggestion-box): This activity titled “Youth Suggestion Box” using Laravel is an individual mini-system that allows users to submit, view, update, and delete suggestions from the youth. It is a web application designed to collect and manage youth feedback and ideas in an organized way. The project demonstrates core Laravel concepts such as database configuration, migrations, routing, controller logic, Blade templating, and full CRUD operations. It provides a structured system where users can add new suggestions, view all submitted entries, see detailed information, edit existing data, and delete records through a clean and functional interface.

Activity 4(recipe-sharing-form): “Recipe Sharing Form” using Laravel is an individual mini-system that allows users to submit and share their own recipes through a web form. It enables users to input recipe details such as recipe name, ingredients, cooking time, origin (e.g., Philippines, Japan, and other countries), and cooking instructions. The system demonstrates Laravel form handling, routing, controller logic, and input validation. It also includes proper error display using Blade templates to ensure complete and valid recipe submissions.

Activity 5(youth-portal): “Youth Portal” using Laravel is a web application that provides user authentication with role-based access for Admin and User accounts. It allows users to register and log in using Laravel Breeze, where Admins and Users are redirected to their respective dashboards. The system also includes its own API and integrates external JSON data showing unemployment rates in the Philippines across different years. It demonstrates Laravel authentication, API creation, and public API consumption, presenting user information, internal data, and unemployment statistics in a single organized dashboard.
